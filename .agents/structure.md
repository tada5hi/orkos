# Project Structure

Single-package TypeScript library. ESM-only output.

## Directory Layout

```
orkos/
├── src/                    # Production source code
│   ├── index.ts            # Public API — re-exports all public symbols
│   ├── types.ts            # Interface definitions (IModule, IApplication, ModuleDependency)
│   ├── application.ts      # Core Application class (orchestrator)
│   ├── constants.ts        # ModuleStatus enum
│   ├── define.ts           # defineModule helper for typed module factories
│   ├── resolve.ts          # External module resolution via dynamic import
│   ├── semver.ts           # Semver constraint validation
│   └── error.ts            # ApplicationError class (extends ebec BaseError)
├── test/
│   └── unit/               # Unit tests (one spec per module)
├── dist/                   # Build output (generated by tsdown)
│   ├── index.mjs           # Compiled ESM bundle
│   ├── index.d.mts         # TypeScript declarations
│   └── index.mjs.map       # Source map
├── .github/
│   ├── workflows/
│   │   ├── main.yml        # CI: build + lint + test
│   │   └── release.yml     # Automated release via release-please
│   ├── actions/             # Reusable composite actions (install, build)
│   └── dependabot.yml       # Dependency update config
├── tsconfig.json            # TypeScript config (extends @tada5hi/tsconfig)
├── tsdown.config.ts         # Build tool config
├── eslint.config.js         # ESLint config (extends @tada5hi/eslint-config)
└── commitlint.config.mjs    # Commit message rules
```

## Module Responsibilities

| File | Purpose |
|------|---------|
| `src/application.ts` | `Application` class — module registration, dependency resolution (Kahn's algorithm), lifecycle management (setup/teardown), status tracking, version validation, external module integration |
| `src/types.ts` | `IModule`, `IApplication`, `ModuleDependency`, `ModuleInput`, `ApplicationContext`, `SetupOptions`, `ModuleDefinition`, and `ModuleFactory` types |
| `src/constants.ts` | `ModuleStatus` and `ApplicationErrorCode` enums |
| `src/define.ts` | `defineModule()` helper — creates typed module factories with default options |
| `src/resolve.ts` | `resolveExternalModules()` — resolves npm packages via dynamic import, supports recursive resolution and auto-install |
| `src/semver.ts` | `satisfies()` — validates version strings against semver range constraints |
| `src/error.ts` | `ApplicationError` — extends `ebec` `BaseError` with structured error codes |
| `src/index.ts` | Barrel export — the entire public API surface |

## Key Dependencies

| Package | Purpose |
|---------|---------|
| `eldin` | Dependency injection container (`IContainer`, `Container`) shared across modules |
| `ebec` | Base error class with structured `code` field |
| `@antfu/install-pkg` | Auto-install missing npm packages (used when `autoInstall: true`) |
